Architecture

New DB tables:

tournaments — id, code (TRNY-XXXX), host_name, format, player_count, status, created_at

tournament_matches — id, tournament_id, round_number, match_index, player_names[], winner, room_code, status

New files:

src/lib/tournamentStore.ts — CRUD for tournaments + matches

src/components/play/TournamentLobby.tsx — join + bracket view

src/components/play/BracketView.tsx — visual bracket with live status

src/components/play/TournamentResults.tsx — final standings + champion

src/hooks/useTournamentChannel.ts — Supabase Realtime for bracket updates

Flow: Host creates tournament → players join lobby → host starts → bracket auto-generated → each match creates a game room → winners reported back → next round seeded → finals → champion

Formats: 4-player (2 rounds), 8-player (3 rounds). Single elimination only to start. Swiss/round-robin later.
