<decisions_made>

- **Return-space convention**: V(s) predicts −expected_score (higher=better). Terminal reward R = −final_score.
- **log_prob_old at temperature=1**: Always computed from raw logits regardless of collection temperature — temperature-invariant importance ratios.
- **index_put (non-in-place)** for assembling log_probs_new: in-place `tensor[mask] = lp` severs the autograd graph; `index_put` preserves gradient flow through policy_loss.

<context>
PPO was chosen because REINFORCE plateaued at ~275 avg across multiple runs. The core issue: REINFORCE does one gradient update per data batch and discards it — wasteful and high-variance. PPO reuses each batch for 4 gradient epochs and uses GAE to give each step a better credit signal. This should break the plateau. If it doesn't, MCTS determinization is the next lever.
</context>
